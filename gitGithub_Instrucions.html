Instruções de comandos no git para github

mkdir

Serve para criar uma pasta

cd

Serve para acessar a pasta criada

ls

Serve para lista os arquivos na pasta

git init

Serve para criar um repositório git na pasta escolhida

git add .

Serve para enviar seus arquivos para uma área temporária chamada "Stage"

git commit -m "Texto Explicativo"

Serve para salvar uma nova versão no repositorio git

git branch -m main

Serve para auterar o branch master para main no git, padrão no github, somente uma vez.

git remote add origin git@github.com: /.git

Serve para associa seu projeto no github com o git (SSH)

git push -u origin main

Serve para adicionar repositorio no github, observação este comando é somente para primeira vez.

Para os demais atualizações de repositorios utiliza se somente "git push"

Agora, segue os comando para atualização de repositorios existentes no github

git status

Serve para mostrar como está a situação do arquivo

git add .

Serve para enviar seus arquivos para uma área temporária chamada "Stage"

git commit -m "Texto Explicativo"

Serve para salvar uma nova versão no repositorio git

git push

Serve para adicionar repositorio no github

git log

Serve para visualizar informações de commits no git

git log --decorate

Serve para visualizar informações de commits no git

git shortlog

Serve para visualizar informações de commits por autores no git

git shortlog -sn

Serve para visualizar quantidade de commits por autores no git

git log --graph

Serve para visualizar graficamente a informação de commits por autores no git

git diff

Serve para visualizar as mudanças antes dos commits no git

git diff --name-only

Serve para visualizar somente o nome do arquivo modificado no commits no git

git checkout

Serve para desfazer a ultima modificação no arquivo no git
